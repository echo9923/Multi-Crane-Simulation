# 任务：Module P 的规划与实现

## 背景

- **Module P 的职责（一句话）**：把 Module O 输出的数据集窗口索引和 episode 权威数据转换为可复现、可验证、无时间泄漏的 STGNN / 轨迹预测训练样本，并提供可直接接入 PyTorch 的 Dataset/DataLoader 入口。

- **上游/下游依赖**：
  - **上游依赖（P 消费的数据）**：
    - Module A 的 `DatasetConfig`，尤其是 `windows.input_steps`、`windows.pred_steps`、`windows.stride_steps`、`windows.risk_label_horizons_s` 和采样配置。
    - Module O 输出的 `dataset_manifest.json`、`dataset_summary.json`、`splits.json` 或等价 split manifest、`windows.parquet`。
    - Module L 输出的 episode 权威文件：`trajectories.parquet`、`pair_risks.parquet`、`graph_edges.parquet`、`tasks.parquet`、`episode_summary.json`、`episode_manifest.json`。
    - Module K 写入或由 `pair_risks.parquet` 承载的离线风险标签字段。
  - **下游被谁调用**：
    - 训练入口脚本，例如 `scripts/build_stgnn_dataset.py`、`scripts/train_stgnn.py` 或后续论文实验脚本。
    - 研究代码中的 PyTorch `Dataset` / `DataLoader`。
    - 后续模型评估、风险分布统计、样本可视化或诊断工具。

- **必须遵守的对外契约**：
  - 只读取 Module O 的 split/window 结果，不重新随机划分 train/val/test，不允许按滑动窗口重新打散导致 episode、scenario、layout 或 num_cranes 泄漏。
  - 每个样本必须来自单个 episode 内的连续窗口，输入窗口不得读取预测窗口之后的信息。
  - `Y_traj` 和 `Y_risk` 只能来自权威轨迹和离线标签，不重算、篡改或伪造训练真值。
  - 可变塔吊数量必须有明确策略：第一版可以通过 mask/padding、按 `num_cranes` 分桶或自定义 collate 处理，但策略必须文档化并测试。
  - 输出样本必须保留可追溯 metadata：`dataset_id`、`split`、`scenario_id`、`episode_id`、`start_frame`、窗口参数、源文件路径或源文件引用、schema version。
  - 转换过程必须做 schema/version/字段完整性校验；不兼容输入要报 conversion error，不能静默生成错误 tensor。
  - 完整 API key、Authorization header、provider secret、raw secret 字符串不得进入训练样本、metadata、summary、日志或错误 payload。

- **明确不属于 Module P 的部分（non-goals）**：
  - 不运行仿真、不批量生成 episode、不做 dataset split；这些属于 Module J/M/O。
  - 不实现塔吊物理、任务状态机、在线风险、安全干预、LLM provider、前端渲染或后端 REST 路由主体。
  - 不计算离线风险真值；离线标签由 Module K 负责，P 只消费标签。
  - 不修改原始 episode 文件、Module O dataset 目录中的权威 manifest/summary/split/window 索引。
  - 不把前端 display-only 距离或风险写入训练真值。
  - 不强制实现完整模型训练算法；第一阶段重点是训练数据转换、Dataset 接口、统计与可测试性。

- **技术栈与目录**：
  - 语言：Python 3 + Pydantic + PyArrow/Parquet + NumPy + pytest。
  - PyTorch 相关入口：优先把核心转换逻辑与 PyTorch 依赖解耦；`torch` 只在 Dataset/DataLoader 适配层中使用。
  - 建议实现目录：`backend/app/training/`、`backend/app/schemas/training.py`、`scripts/build_stgnn_dataset.py`，必要时少量对接现有 dataset schema。
  - 测试目录：`backend/app/tests/test_stgnn_*.py`、`backend/app/tests/test_moduleP_*.py`。
  - 文档目录：`docs/moduleP/`
