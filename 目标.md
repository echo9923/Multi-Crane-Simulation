  # 任务：Module O 的规划与实现

## 背景

- **Module O 的职责（一句话）**：从多个仿真 episode/run 中批量构建可复现、可审计、无泄漏的训练数据集，并输出 dataset split、质量门禁、窗口索引、dataset manifest 和 dataset_summary。

- **上游/下游依赖**：
  - **上游依赖（O 消费的数据）**：
    - Module A 的 `DatasetConfig`、`ScenarioConfig`、`ExperimentConfig`、`ResolvedConfig`。
    - Module J/M 的本地 runner 或 episode service，用于批量生成 episode。
    - Module L 的 run 目录、`episode_summary.json`、`episode_manifest.json`、`trajectories.parquet`、`pair_risks.parquet`、`graph_edges.parquet`、`tasks.parquet`、`weather.parquet`、日志和 replay 文件。
    - Module K 写入或提供的离线风险标签字段。
  - **下游被谁调用**：
    - `scripts/batch_generate.py` 和 `scripts/build_dataset.py`。
    - Module M 的 `/datasets` 与 `/datasets/{dataset_id}/summary` 只读查询。
    - Module P 的 STGNN / 轨迹预测数据转换与训练入口。

- **必须遵守的对外契约**：
  - 不按滑动窗口随机切分 train/val/test，必须按 episode/scenario/layout/num_cranes 等不泄漏维度切分。
  - 未通过质量门禁的 episode 只能进入 quarantine，默认不得进入正式 split。
  - `dataset_summary.json` 必须报告目标比例、实际比例、差距、风险等级分布、任务完成率、碰撞/near miss 计数、quarantine 数量、schema version、配置摘要和 git commit。
  - 不修改原始 episode 轨迹、命令、任务、风险或日志；O 只能读取、校验、复制/链接、索引和聚合。
  - 不通过穿模、关闭机械限制或伪造标签提高风险样本比例。
  - 完整 API key、Authorization header、provider secret、raw secret 字符串不得进入 dataset manifest、summary、索引或 API payload。

- **明确不属于 Module O 的部分（non-goals）**：
  - 不实现塔吊物理、任务状态机、在线风险、安全干预、LLM provider、前端渲染或后端 REST 路由主体。
  - 不计算离线风险真值；离线标签由 Module K 负责。
  - 不写单 episode recorder 权威表；Module L 负责 episode 级 Parquet/JSONL/manifest/summary。
  - 不训练模型，不实现 PyTorch Dataset 主逻辑；Module P 消费 O 的 dataset 输出。
  - 不把前端 display-only 距离或风险写回训练真值。

- **技术栈与目录**：
  - 语言：Python 3 + Pydantic + PyArrow/Parquet + pytest。
  - 实现目录：`backend/app/data/`、`backend/app/schemas/dataset.py`、`scripts/batch_generate.py`、`scripts/build_dataset.py`，必要时少量对接 `backend/app/api/cli.py` 和 Module M 的 dataset root 配置。
  - 测试目录：`backend/app/tests/test_dataset_*.py`、`backend/app/tests/test_moduleO_*.py`。
  - 文档目录：`docs/moduleO/`
